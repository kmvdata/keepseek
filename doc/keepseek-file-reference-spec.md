# KeepSeek 文件引用语法规范

本文档定义 KeepSeek 输入组件中唯一允许的文件引用高亮语法。输入组件据此识别引用并将其渲染为可点击的文件链接。

---

## 一、基本语法

所有文件引用**必须**使用半角尖括号 `<` `>` 包裹，格式如下：

```
<文件路径#L行号范围>
```

其中：
- **路径**：工作区相对路径或绝对路径
- **`#`**：路径与行列之间的分隔符
- **`L`**：行号标识（大写）
- **`C`**：列号标识（大写，可选）

---

## 二、三种引用形式

### 2.1 引用整个文件（无行列范围）

```
<文件路径>
```

**示例：**

```
<src/config.ts>
<docs/README.md>
```

### 2.2 带行号范围

```
<文件路径#L起始行-L结束行>
```

**示例：**

```
<src/config.ts#L81-L87>
<signal_tracker_py/app/etc/develop/config.yaml#L81-L87>
```

### 2.3 带行列范围

```
<文件路径#L起始行C起始列-L结束行C结束列>
```

**示例：**

```
</Users/kermit/Projects/dabai/signal_tracker/signal_tracker_py/app/etc/develop/config.yaml#L81C1-L87C1>
<src/utils.ts#L10C5-L10C20>
```

---

## 三、语法规则

| 规则 | 说明 |
|---|---|
| **包裹符号** | 必须使用 `<` `>`，方括号 `[` `]` 或圆括号 `(` `)` 不被识别 |
| **路径格式** | 支持绝对路径（`/home/...`、`C:\...`）和工作区相对路径（`src/main.ts`）；路径中不可包含 `<`、`>`、换行符 |
| **行标识** | 大写 `L`，行号从 1 开始 |
| **列标识** | 大写 `C`（可选），列号从 1 开始 |
| **范围符** | 半角连字符 `-` 连接起止位置 |
| **分隔符** | 路径与行列之间使用 `#` |
| **独立成行** | 引用应放在单独一行，避免与其他文本混合 |
| **真实文件** | 只引用工作区中实际存在的文件，不虚构路径 |

---

## 四、反例（不会被高亮）

| 错误写法 | 原因 |
|---|---|
| `[config.yaml(L81-L87)]` | 方括号不被识别 |
| `(config.yaml#L81-L87)` | 圆括号不被识别 |
| `<config.yaml:81-87>` | 缺少 `#L` 前缀 |
| `配置见 <config.yaml#L81> 继续` | 引用不在独立行 |

---

## 五、正例

```
问题分析如下：

<src/auth/login.ts#L45-L72>

上述代码中，第 52 行的逻辑存在并发问题，建议修改为：

<src/auth/login.ts#L52C1-L52C45>
```

---

## 六、后端识别逻辑（参考）

后端使用以下正则表达式匹配引用：

- 主匹配：`/<([^<>\n]+)>/gu`
- 行列解析：`FILE_REFERENCE_LINE_PATTERN` 解析 `#L行号C列号` 格式

前端渲染时将匹配结果转换为 `<a class="rich-file-link">` 可点击链接，序列化时输出为：

```
标签文本
<路径#行号>
```
